# Codex Usage Cheat Sheet

이 문서는 사용자가 Codex를 **Study Agent처럼 쓰기 위한 짧은 명령 예시**다.

---

## 1. Day 시작 명령
```text
Day 3 시작
```

```text
Day 10 시작. 지금 코드 기준으로 필요한 선행 작업이 있으면 한 개만 먼저 알려줘.
```

---

## 2. 다음 단계만 받고 싶을 때
```text
다음 단계
```

```text
한 번에 많이 말하지 말고, 다음 한 단계만 알려줘.
```

---

## 3. 설명 먼저 받고 싶을 때
```text
지금 단계에서 왜 이걸 하는지 먼저 설명해줘. 코드 말고 개념 먼저.
```

```text
오늘 꼭 알아야 할 개념 3개만 알려줘.
```

---

## 4. 코드 리뷰 요청
```text
코드리뷰 해줘. 전체 재작성 말고 문제점만 알려줘.
```

```text
구조적으로 이상한 부분이 있는지, 테스트가 부족한지 중심으로 봐줘.
```

---

## 5. 테스트 코드 같이 짤 때
```text
테스트 코드 같이 짜줘. 먼저 무엇을 검증해야 하는지부터 설명해줘.
```

```text
이 기능에 대한 Integration Test를 Given/When/Then 형식으로 같이 만들어줘.
```

---

## 6. CS 지식 연결해서 공부할 때
```text
CS 설명해줘. 지금 단계와 연결해서 설명해줘.
```

```text
트랜잭션이 왜 필요한지 GiftFlow 주문 생성 흐름 기준으로 설명해줘.
```

---

## 7. 실무 관점 질문
```text
실무에서는 이걸 왜 이렇게 구현하는지 설명해줘.
```

```text
이 구조에서 장애가 나면 운영자는 어디부터 봐야 해?
```

---

## 8. 학습 확인용 질문 받기
```text
내가 이해했는지 확인 질문 3개만 내줘.
```

```text
퀴즈 내줘. 이론 1개, 코드/설계 1개, 실무 1개.
```

---

## 9. 막혔을 때
```text
막혔어. 지금 단계에서 필요한 해결책만 알려줘.
```

```text
이 에러 원인 후보 3개만 좁혀줘. 앞 단계 전체 리팩토링 말고 최소 수정으로.
```

---

## 10. 공부 리듬 유지용
```text
오늘 여기까지 했어. 내가 뭘 이해했는지 체크해줘.
```

```text
오늘 작업 기준으로 회고를 도와줘. 잘한 점 / 부족한 점 / 다음 액션 1개만.
```
